# Задание 4: Property-based тестирование с testing/quick

## Что такое property-based тестирование?

Вместо проверки конкретных примеров (example-based testing), мы проверяем свойства,
которые должны выполняться для ЛЮБЫХ входных данных.

## Основные концепции:

### 1. `quick.Check(property, config)`

- Автоматически генерирует случайные входные данные
- Проверяет свойство для каждого набора данных
- Останавливается при первой неудаче

### 2. Свойство (property)

Это функция, которая возвращает `bool`. Например:

```go
func commutativeProperty(a, b int) bool {
    return a + b == b + a
}
```
